About

Intro

At PMSIpilot, we migrated from SVN to Git in late 2010, as the need for a more powerful VCS was rising. Now that we had a great DVCS tool in our hands, we needed to work on a branch model to use within our dev teams (about 15 people).

We thought (and still think) it would to be easier to abstract this branch model with a powerful CLI tool to use on a daily basis. Inspired by git-flow, we developed pflow, a CLI tool we hope you'll like and use as we have been doing for more than one year :)

Concept

With the powerful Git merging capabilities, whenever we start to develop something (feature, bug fix...), we do it on a new Git branch to isolate it.

In pflow langage, we call it a feature.

The goal of pflow is to provide commands to interact with features:

  • start
  • publish
  • sync
  • finish
  • review request
  • ...

Installation

PHP >= 5.2 and Git >= 1.7 must be installed.

Now follow these instructions: 

git clone git://github.com/pmsipilot/pflow.git
cd pflow
ln -sf `pwd`/bin/git-pflow ~/bin/git-pflow
ln -sf `pwd`/bin/pflow ~/bin/pflow
Info We suppose here that ~/bin is in your PATH. Just change the previous ln -sf commands to adjust them if it is not the case.

You should now have pflow installed: 

pflow --version
your_pflow_version

Configuration

pflow is installed, let's use it in a Git project. The minimal configuration should look like this: 

cd my_project
pflow init
-!- Jabber output disabled
PMSIpilot flow configuration
-!- PMSIpilot flow can work with Crew
handle Crew stuff (yes|no) [no] : 
-!- fetch origin
team branch name : master
-!- branch -a
-!- branch -a
-!- PMSIpilot flow can automatically create a database when you start a branch (using the -d option of feature start)
handle database stuff (yes|no) [no] : 
PMSIpilot can announce branch starts and merges on jabber. You will need your jabber username and password.
announce stuff on jabber (yes|no) [no] : 
thank you, pflow is now configured!

Explanations:

  • Crew: pflow can be used to request code reviews with Crew, another tool we developped :)
  • Team branch name: the default base branch for each of your feature starts. Branch must be available on origin remote.
  • Handle database stuff: as well as the code of every feature is isolated in a Git branch, your data could be isolated too. pflow can create a MySQL schema when you start your feature with -d flag.
  • Jabber stuff: pflow can talk to a jabber server to tell when you start/publish/finish a feature.

Commands

Detailed description for each pflow command is coming soon :)

You can look at the available commands by just running pflow in your console:

pflow
  abort                        tries to abort the last interrupted action

  continue                     tries to continue the last interrupted action

  continue reset               resets continue informations

  feature finish               merges a feature branch in the base branch
    -p                         push the changes after merging
    -d                         drop database for feature branch

  feature info                 show some informations about the feature

  feature publish              pushes the feature branch on the origin

  feature start <feature_name> start a feature branch off the base branch
    -b branch                  base branch from feature rather than the default base branch
    -d                         create database for feature branch
    -n                         do not update the base branch before branching
    -p                         automatically push the branch after creation

  feature sync                 synchronizes feature branch with base branch and remote branch

  feature sync base            synchronizes feature branch with its base branch

  feature sync remote          synchronizes feature branch with its remote branch

  init                         initialize PMSIpilot flow
    -d                         remove all PMSIpilot flow configuration
    -s section                 section to configure

  log purge                    purges the log in .git/pflow/history

  review request               create a review request on Crew server for current branch

  review status                print review status of the current branch

  state                        shows infos on the current state of your repository

  update                       updates pflow (requires writing privileges to install prefix)
  
  global options:
    --no-hook                  if passed, pre/post command hooks will not be executed
